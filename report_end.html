Explanations:

Red status

 CRITICAL: no backup has been performed

Orange status

 WARNING: backup does not work appropriately
“Conflicted!!” A backup has been performed, but a ANR2579E has been thrown, erroneously indicating “no backup”
Total [MB] logical occupancy of server disk
(since the server uses “deduplication”, it’s impossible to say how much space the client actually uses)
# FS Number of file spaces that have been backed up (“Filespace” = file system)
Version The version of the backup client.
Latest versions: Linux-RPM: LINUXX86VER; Linux-DEB: LINUXX86DEBVER; macOS: MACOSVER; Windows: WINDOWSVER

 

Click on the name of a client to see more details about it. Columns are sortable
